Cahier des charges Discorder

Présentation du Projet :

  • Nom : Discorder
  • Description : Nouveau Discord offrant de nombreuses fonctionnalités indisponibles actuellement.
  • Objectif : Faciliter l'utilisation contournant certaines restrictions.
  • Fonctionnalités : Divers moyens d'interaction avec Discord.
  • Public cible : Utilisateurs quotidiens de Discord.
  • Avantages : Semi-gratuit, sécurisé, facile à utiliser, facilité de création de scripts (plugins, self-bot, bot Discord).

Planification du Projet :

Interface Style :

  • Design dans le style de Discord.
  • Utilisation de couleurs contrastées et de polices lisibles.
  • Adaptation aux différents formats d'écran (responsive).

Interface Modules :

Les modules doivent être à la place des serveurs dans l'application Discord.

1er module :

  • Page de présentation avec les actualités du projet et des conseils d'utilisation.

2e module :

  • Une réplique de l'interface de base de Discord (avec ajout d'une fonction pour dupliquer cette interface pour plusieurs comptes, par exemple 10).

3e module :

  • Ajout d'un menu pour les fonctionnalités avec l'API classique (voir le début de Fonctionnalités).
  • Ajout d'une option pour choisir sur quel compte exécuter les scripts.

4e module :

  • d'une sorte de marché gratuit de plugins pour les utilisateurs.
  • Activation et désactivation des plugins.
  • Documentation pour la création de plugins.

5e module :

  • Création de self-bots avec 3 modes :
    • Mode 1 : Self-bot préfait avec simplement des cases à cocher.
    • Mode 2 : Self-bot à construire avec des blocs de code, un peu comme Scratch.
    • Mode 3 : Liste des fonctions prêtes avec l'API.
  • Un sous-module pour les modes 1 et 2 avec la possibilité de lancer le self-bot sur plusieurs comptes Discord si nécessaire.
  • Un sous-module pour le mode 3 demandant l'emplacement du fichier du self-bot sur plusieurs comptes Discord si nécessaire.

6e module :

  • Création de bots Discord en Python avec 3 modes :
    • Mode 1 : Bot Discord préfait avec simplement des cases à cocher.
    • Mode 2 : Bot Discord à construire avec des blocs de code, un peu comme Scratch.
    • Mode 3 : Liste des fonctions prêtes avec l'API.
  • Un sous-module pour les modes 1 et 2 avec la possibilité de lancer le bot Discord.
  • Un sous-module pour le mode 3 demandant l'emplacement du fichier du bot Discord en question.

7e module : Interface

  • Sous-module permettant d'activer les thèmes.
  • Création de thèmes.
  • Importation de thèmes grâce à un marché.

8e module : Paramètres

  • Activation des modules + avertissement des risques de bugs.

Fonctionnalités :

  • Obtenir les informations d'une ID avec l'API Discord.
  • Obtenir les informations d'un token avec l'API Discord.
  • Obtenir ses propres informations avec l'API Discord.
  • Obtenir les informations d'une guilde avec l'API Discord.
  • Obtenir ses propres informations de paiement avec l'API Discord.
  • Obtenir les informations de paiement d'un token avec l'API Discord.
  • Supprimer des messages avec l'API Discord.
  • Envoyer des messages directs à un serveur grâce à une ID de guilde.
  • Effectuer des spams dans un canal.
  • Effectuer un spam complet sur un serveur.
  • Pouvoir voir les messages supprimés.
  • Avoir plusieurs comptes Discord simultanément depuis le client.
  • Modifier l'interface du client sans connaissance de code.
  • Créer ses propres self-bots sans connaissance de code.
  • Créer un langage spécifique à la création de self-bots.
  • Créer ses propres bots Discord sans connaissance de code.
  • Créer ses propres plugins.
  • Créer un langage spécifique à la création de plugins.
